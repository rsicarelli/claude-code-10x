# Writing Style

## Gender-neutral language (PT-BR)
- Use "dev", "você" (direct address), "quem desenvolve"
- NEVER use "o desenvolvedor", "a desenvolvedora", "o engenheiro", "o operário"
- For roles in the factory analogy, use verbs: "quem opera", "quem projeta"

## PT-BR natural tone
- Use contractions: pra, pro, num, dum, tá, né
- Write like you talk to a colleague, not like a textbook
- Author voice: opinions, humor, first person ("eu") when natural
- Vary sentence length. Mix short punchy sentences with longer flowing ones.

## Technical terms
- Keep in English on first use: agentic, prompt engineering, context engineering, harness engineering, context window, token, tool use, function calling, vibe coding, autocomplete, code completion, MCP, harness, loop
- On first use, optionally explain in parentheses in Portuguese: "**agêntica** (ou seja, capacidade de agir por conta própria)"
- After first explanation, use freely without re-explaining

## Anti-patterns to AVOID
- **Em-dash-as-crutch**: Don't use "X — inline explanation" to shoehorn context. Use periods, colons, commas, or parentheses instead.
- **Formulaic tags**: Don't repeat a structural tag like "Na fábrica:" before every analogy. Weave analogies naturally.
- **Symmetric parallelism**: Don't write 3 sections with identical structure (same opening pattern, same closing pattern). Vary the form.
- **Stat-dumping**: Don't cluster 3+ data points in one paragraph. Distribute data as punchlines throughout the text.
- **Press-release tone**: Don't describe tools like product announcements. Be conversational.
- **Bullet-point recap conclusions**: Don't end with a bulleted list summarizing what was already said. Close with a forward-looking thought.
- **Excessive final punctuation**: Don't write 5+ consecutive short sentences ending with periods. Combine with commas and connectors.
